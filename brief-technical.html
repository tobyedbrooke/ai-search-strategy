Technical Team Brief

Make Trade Republic readable by AI

Three technical changes that determine whether AI systems can read and cite our content. Based on confirmed audit findings from the TR support page and main site.

Why this matters. AI systems like ChatGPT, Perplexity and Google AI Overviews fetch pages the same way a simple HTTP client does — they do not execute JavaScript. If a page's content is fetched by JS after load, the AI sees an empty shell. We have confirmed this is exactly what is happening with TR's support page. Additionally, without structured data markup (schema), AI has to guess what TR is, what products cost, and which text answers which question. These three changes fix both problems.
Confirmed audit findings
Finding 1 — Main site is fine; the accordion HCAs are the problem

The main TR website is already served correctly — AI crawlers can read the homepage and product pages. The problem is specific to the Help Centre accordion articles (HCAs): their content is fetched from the Strapi CMS via a separate API call after the page loads, so AI sees the accordion headings but none of the answers inside them.

This means: when someone asks Perplexity "how do TR fees work?" or "what is TR's interest rate?", the AI cannot cite TR's own support page — even though TR has accurate, well-written answers published there. The answers are invisible to every crawler.

Finding 2 — FAQPage schema exists but loads at the same time as the content

TR already has FAQPage schema markup on the support page. The problem is that it is injected client-side alongside the HCA content, so it loads after the crawler has already left. Fixing the HCA rendering (Task 1) makes the existing schema readable automatically — no schema work is required separately.

Finding 3 — Some HCAs may only exist in-app, not on the web

Any Help Centre article that exists only inside the TR app cannot be indexed or cited by AI. An audit is needed to identify which articles are web-visible and which are app-only, and to publish missing articles to the support page.

Tasks
1
Make the Help Centre accordion articles server-side rendered Short term — highest priority

The main TR site is already SSR — this change is scoped only to the HCA content inside the support page accordions. Currently their content is fetched from the Strapi CMS API client-side after page load. The fix moves this fetch to run server-side, so the answers are embedded in the initial HTML that crawlers receive.

The change: Replace the client-side fetch with useAsyncData or useFetch called at the top level of the component setup (not inside onMounted). Nuxt will run this on the server and inline the data into the HTML.

// Before (client-only — invisible to AI) onMounted(async () => { const data = await fetch('/api/support-articles') articles.value = data }) // After (server + client — AI can read it) const { data: articles } = await useAsyncData('support-articles', () => $fetch('/api/support-articles') )

FAQPage schema is also fixed by this change. The schema already exists but loads client-side alongside the content. Once the fetch moves server-side, the schema becomes readable automatically — no separate schema work is needed.

How to verify:

curl -s https://traderepublic.com/en-de/support | grep -c "What fees do you charge"

If the count is 1 or more, the content is in the HTML and AI can read it. Also run the URL through Google's Rich Results Test — if FAQPage schema is detected, both the content and schema are working.

2
Audit which HCAs exist only in-app and publish them to the web Short term

Any Help Centre article that exists only inside the TR app cannot be indexed or cited by AI — it is invisible to every crawler regardless of how useful or accurate it is. If a user asks Perplexity a question that TR answers in-app but not on the web, AI will source the answer from a third party.

Action: Compare the full list of in-app HCAs against what is published on the web support page. For every article that exists in-app but not on the web, publish it to the support page. Prioritise articles that answer high-frequency AI queries:

  • →Fees and pricing
  • →Safety and regulation
  • →Interest rate and cash
  • →Card usage
  • →Account closure and KYC
  • →How to contact support
3
Add Organisation and Product schema to the homepage and product pages Medium term

The homepage is already SSR (Nuxt renders it server-side). It just lacks schema. Adding Organization and Product schema here is low-effort — no rendering change needed, only adding a <script type="application/ld+json"> block to the page head.

Homepage — Organization schema: Establishes TR's identity as a machine-readable fact — name, regulation, markets, licence type. AI uses this to answer "what is Trade Republic?" and "is TR regulated?"

{ "@context": "https://schema.org", "@type": "FinancialService", "name": "Trade Republic", "legalName": "Trade Republic Bank GmbH", "url": "https://www.traderepublic.com", "foundingDate": "2015", "areaServed": ["DE","AT","FR","ES","IT","NL","BE","PT","FI","PL","..."], "regulatoryBody": { "@type": "Organization", "name": "BaFin" }, "memberOf": { "@type": "Organization", "name": "German Deposit Guarantee Scheme" } }

Product pages — Product schema: Add to the Interest Account, ETF Savings Plan, and Stock Trading pages. Gives AI current rates and fees to quote directly rather than inferring from stale editorial content.

  • →Interest Account — rate: 2.00% p.a. (must update whenever ECB rate changes — stale rate in schema is worse than no schema)
  • →ETF Savings Plan — fee: €0, minimum: €1/month
  • →Stock/ETF trading — trade settlement fee: €1 per trade, savings plans: €0

Rate update process: Assign one owner to update the interest rate schema whenever the ECB changes the rate. This should be part of the existing rate-change communications process — add schema update as a checklist item.

Expected outcome After Tasks 1 and 2: TR's support page content becomes crawlable for the first time. Perplexity and Google AI Overviews can read and cite TR's own answers to fee, safety, and product questions — rather than relying on third-party editorial sites. After Task 3: AI quotes current TR rates accurately, with TR as the primary source.
